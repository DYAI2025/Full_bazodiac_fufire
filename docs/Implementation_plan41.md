Die Uebersichtsseite wird von einer datenreichen Chart-/Auditseite zu einem klaren Einstieg in die persoenliche Fusion-Signatur. Der User soll sofort verstehen: Was ist mein Muster, was traegt, was reibt, was hilft heute und wo kann ich sinnvoll vertiefen. Das Zodiac Wheel wird dabei zum lebendigen visuellen Identitaetsobjekt, ohne Datenwahrheit oder Pruefbarkeit zu verlieren.  Scope. Primaer betroffen: public/src/pages/OverviewPage.js, public/src/components/NatalChartWheel.js, public/src/domain/overviewModel.js, relevante Styles in public/src/styles/**, Tests unter test/**, Playwright Specs unter test/e2e/**, QA-Dokumentation unter docs/qa/**. Backend-Architektur und FuFire-Berechnungslogik bleiben unveraendert.  Bedingungen (hart).  TDD-first: jede fachliche Aenderung beginnt mit einem fehlschlagenden Test.  Keine hardcodierten astrologischen Zielwerte und keine Fake-0-Grad-Fallbacks.  Keine neuen astrologischen Kernberechnungen im Frontend, nur Visualisierung vorhandener Daten.  Abschluss nur mit Playwright-Live-Browser-Test, Screenshots, optischem Review und Code Review.  Detaildaten bleiben erreichbar, duerfen aber die Overview nicht dominieren.  Akzeptanzkriterien.  Overview beantwortet im ersten Screen: Wer bin ich im Modell, was ist die Hauptspannung, was kann ich als naechstes tun?  Hero zeigt grosses, plastischeres Wheel plus Fusion-Signatur und drei Evidence Cards.  Element-Oekonomie zeigt keine internen Feldnamen wie Distribution, Dominant, Deficient, Plan, Properties, TodayLever.  Wheel zeigt Zeichenring, Haeuser, AC/MC/IC/DC, Planeten, Top-Aspekte und eine auditierbare Datenliste.  Default-Ansicht zeigt maximal Top-3 Aspekte; alle Details liegen in Tabs/Accordions.  Explizit out-of-scope.  Kein Backend-Refactor.  Keine DB- oder API-Vertragsaenderung.  Kein kompletter Rebuild der App-Navigation.  Keine V3-Signatur als Ersatz fuer das Zodiac Wheel.  Keine neue externe Chart-Library ohne separate Entscheidung.  Done-Definition. Alle Overview-Tests, npm test, Playwright Desktop/Mobile, Wheel-Hover-Screenshots, Code Review und PO-Screenshot-Review sind gruen; QA-Bericht liegt unter docs/qa/.
